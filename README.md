# Trail

# AI-Enabled Progress Tracker + Notion AI Agent + Smart Work Planner

## One system to track, plan, and resume every project you own.

---

## What Is This?

You have dozens of projects across GitHub and Notion. You start them, pause them, forget where you were, lose hours context‑switching, and miss deadlines.

**This system fixes all of that.**

It connects directly to your GitHub repos and Notion databases, then:

- **Tracks progress** – automatically measures completion % from commits and task status.
- **Generates resumption reports** – “You stopped at `auth.py` line 142. Remaining: 1.2 hours.”
- **Plans your day** – schedules work across multiple projects respecting deadlines, priorities, and your energy patterns.
- **Verifies actual work** – if you didn’t commit as planned, it reassigns the task (even partial progress).
- **Learns from you** – estimates get more accurate over time.
- **Listens anywhere** – use natural language in CLI or directly inside Notion with `@ai`.

It’s like a second brain for your work that never forgets.

## Key Features

| Feature | What It Does |
|---------|---------------|
| **One project = one identity** | Every project is locked to exactly one GitHub repo and one Notion database. No cross‑pollution. |
| **Automatic sync** | Hourly (or webhook) fetch of commits, PRs, issues, Notion tasks, sub‑tasks, dependencies. |
| **Resumption reports** | “What was done, when, what’s left, where to pick up” – generated by AI with source citations. |
| **Smart work planner** | Plans your week based on deadlines, parallel project limit, constant project, working hours, holidays, meetings. |
| **Partial progress detection** | Sees you made 2 commits but task isn’t done → estimates remaining work (e.g., 40% done, 1.2h left). |
| **Auto‑reassignment** | At end of day, incomplete tasks are rescheduled into the next available slot. |
| **Cross‑project dependencies** | Task B in Project B can wait for Task A in Project A. Planner respects the order. |
| **Context switch penalty** | Learns how long you lose switching between specific projects and adds buffers. |
| **Calendar integration** | Reads Google/Outlook meetings, auto‑blocks time, adjusts plan. |
| **Notion AI Agent** | Type `@ai what should I work on?` inside any Notion page – gets answered. |
| **AI Brain** | Conversational interface (CLI or Notion) that remembers past conversations and explains decisions. |
| **Cost‑aware LLM routing** | Uses local Llama 3 for cheap tasks, Claude only for complex reasoning – stays under $10/month. |
| **Gamification** | Points, streaks, badges for following your plan. |

## Quick Start (MVP in 4 Weeks)

You don’t need to build everything at once. Here’s the fastest path to a working system:

### Week 1 – Foundation
- Set up PostgreSQL and Redis.
- Create the `projects`, `commits`, `notion_tasks` tables.
- Write a script that fetches commits from one GitHub repo and stores them.

### Week 2 – Add Notion
- Fetch tasks from one Notion database.
- Implement basic linking (commit message contains task ID).

### Week 3 – Basic Planner
- Hardcode a rotation schedule (e.g., Project A every day, Project B every other day).
- Print a daily plan to the terminal.

### Week 4 – CLI & Verification
- Add a command: `tracker status` shows last commit and task progress.
- At end of day, compare planned vs actual commits → print summary.

**You now have a usable MVP.** Then add features incrementally: AI reports, auto‑reassignment, Notion AI agent, calendar sync, etc.

## Installation (Once Built)

```bash
# Clone the repository
git clone https://github.com/yourname/progress-tracker.git
cd progress-tracker

# Set up environment
python -m venv venv
source venv/bin/activate  # or `venv\Scripts\activate` on Windows
pip install -r requirements.txt

# Configure secrets (GitHub token, Notion token, OpenRouter key)
cp .env.example .env
# edit .env with your tokens

# Initialize database
tracker db init

# Add your first project
tracker project add \
  --name "Auth Service" \
  --key "AUTH-01" \
  --github "github.com/you/auth-service" \
  --notion-db "abc123..." \
  --deadline "2026-05-15"

# Run initial sync
tracker sync --project AUTH-01

# See your plan for today
tracker plan today

# Talk to the AI Brain
tracker brain "What should I work on right now?"
```

## Configuration Example

Create `~/.tracker/config.yaml`:

```yaml
user:
  work_start: "09:00"
  work_end: "17:00"
  lunch_start: "12:00"
  lunch_end: "13:00"
  deep_work_minutes: 120
  max_parallel_projects: 2
  constant_project: "AUTH-01"
  working_days: [1,2,3,4,5]  # Mon-Fri
  timezone: "America/New_York"

llm:
  budget_monthly_usd: 10
  default_model: "openrouter/anthropic/claude-3.5-sonnet"
  cheap_model: "openrouter/google/gemini-2.0-flash"
  local_model: "ollama/llama3"

calendar:
  google: true
  outlook: false

abandonment:
  warning_days: 7
  critical_days: 14
  archive_days: 21
```

## Commands Overview (CLI)

| Command | Description |
|---------|-------------|
| `tracker project add` | Register a new project (GitHub + Notion) |
| `tracker sync` | Force a sync of all or one project |
| `tracker status [project]` | Show current progress, last commit, next task |
| `tracker report [project]` | Generate detailed resumption report |
| `tracker plan today` | Show today’s schedule |
| `tracker plan week` | Show weekly plan |
| `tracker brain "question"` | Ask the AI Brain anything |
| `tracker log --project X --hours 2` | Manually log time |
| `tracker dependency add --task X --blocks Y` | Add a dependency |
| `tracker archive [project]` | Archive a finished project |

## Notion Integration

Inside any Notion page (in a database tracked by the system), type:

```
@ai what is the status of Project A?
@ai summarize this page
@ai generate subtasks for "Implement JWT"
@ai move this task to Done
@ai why was my task reassigned yesterday?
```

The AI Brain will answer directly below your command as a callout block.
